[ ] Docker Compose
    - Tutti i container sono UP: sentiment-app, prometheus, grafana, node-exporter
    - Controlla con: docker-compose ps

[ ] Reti Docker
    - Tutti i servizi sulla rete 'monitoring'
    - Controlla con: docker network inspect monitoring

[ ] Prometheus
    - File prometheus.yml esiste e si trova in una cartella montata correttamente
    - Target configurati con nomi dei servizi (es. sentiment-app:5000, node-exporter:9100)
    - Controlla target UP: http://localhost:9090/targets
    - Eventuali reload config se modificato prometheus.yml

[ ] Flask App (sentiment-app)
    - Endpoint health funzionante: http://localhost:5000/health
    - Endpoint metriche funzionante: http://localhost:5000/metrics
    - Controlla log container: docker logs sentiment-analysis-project-sentiment-app-1

[ ] Node Exporter
    - Porta 9100 esposta e accessibile da Prometheus
    - Controlla log container: docker logs sentiment-analysis-project-node-exporter-1

[ ] Grafana
    - Porta 3000 accessibile: http://localhost:3000
    - Credenziali corrette (default admin / admin123)
    - Dashboard configurata per leggere Prometheus sulla rete 'monitoring'
    - Controlla log container: docker logs sentiment-analysis-project-grafana-1

[ ] Metriche generali
    - sentiment_requests_total > 0 quando invii richieste
    - sentiment_request_duration_seconds e sentiment_errors_total aumentano correttamente
    - Verifica in Prometheus con query semplici (es: sentiment_requests_total)

[ ] Build e riavvio
    - Se modifiche a Dockerfile o prometheus.yml → docker-compose build + docker-compose up -d
    - Per riavvio veloce senza rebuild → docker-compose restart

[ ] Troubleshooting
    - Ping tra container: docker exec -it <container> sh → ping <altro_servizio>
    - Controllo log errori in caso di target DOWN o Flask non raggiungibile
